# Legacy To Remaining Phases Map

Status: mapa de oportunidades do app piloto para fases futuras.

## Fase 8 - Memoria

- Templates como modelos governados.
- Listas estruturadas por template, nao planilha geral.
- Caixola como incubacao e recuperacao, nao Inbox.
- Anexos privados com metadata.
- Busca com origem, entidade, status e acao possivel.
- Vinculos contextuais separados de dependencias.
- Arquivados como estado recuperavel.
- Concluidos como historico operacional, nao lixo.

Nao fazer: file manager, wiki, knowledge base generica, storage publico ou dump infinito.

## Fase 9 - Idea

- Structured actions.
- Context builder minimo por superficie.
- Chat contextual como drawer, nao rota principal.
- Output schema validado.
- `requires_confirmation` obrigatorio para acao persistente.
- Safety Gate antes de criacao, conversao, replanejamento, vinculo ou exclusao.
- Logs minimizados.
- Incerteza e lacunas declaradas.

Nao fazer: `IntelligencePage` como rota principal, prompt amplo, acao direta ou logs com texto bruto por padrao.

## Fase 10 - Entity Sheets

- Familia de sheets por entidade.
- Metadata por entidade traduzida em campos especificos.
- Project Stages.
- Milestones.
- Riscos.
- Event Prep.
- Anexos.
- Vinculos.
- Dependencias.
- Composicao.
- Historico.
- Edicao inline controlada.
- Campos derivados nao editaveis.

Nao fazer: copiar `EntitySheet.tsx`, mega-formulario universal ou metadata livre como interface final.

## Fase 11 - Abrir o Dia, Fechar o Dia e Revisao Semanal

- Morning Ritual traduzido para Abrir o Dia.
- Projection propria.
- Replanejamento como operacao de coerencia, nao limpeza de backlog.
- Ordenacao manual temporaria rastreavel.
- Weekly Review como continuidade, direcao e capacidade, nao relatorio produtivista.
- Day Reading curta, operacional e sem tom motivacional generico.

Nao fazer: ritual gamificado, produtividade semanal, ranking ou faxina de tarefas.

## Timeline

- Timed items.
- Untimed items.
- All-day separado.
- Slot creation como possibilidade futura.
- WeekHeatmap como pressao operacional, nao analytics.
- Event Prep como preparacao relativa a evento.

Nao fazer: calendario comum, grafico BI ou duracao inventada.

## Planejar

- Project Stages como subobjeto de Projeto, nao entidade principal.
- Milestones como evidencia de progresso, nao tarefa automatica.
- Risks como sinais de degradacao, nao itens operacionais.
- Goal progress como leitura qualitativa ou metrica explicita, sem falsa precisao.
- Goal health qualitativo, com explicacao de fatores.
- Supports como contribuicao para direcao, nao dependencia.
- Habitos e rotinas conectados a metas, sem streak ou gamificacao.
- PL05 Essenciais protegidos e PL06 Revisao semanal ainda como pendencias planejadas.

Nao fazer: OKR tool, PM tool, score opaco ou backlog de projeto.

## Fase 13 - Central

- Dados.
- Exportacao.
- Exclusao.
- Privacidade.
- IA e transparencia.
- Conta.
- Suporte.
- Delete account governado.
- Exportacao antes de promessa de controle pleno.

Nao fazer: Central como ajuda simples ou Admin disfarçado.

## Fase 14 - Admin

- Admin separado de Central.
- Roles reais.
- Backend e RLS.
- Auditoria.
- Prompt governance.
- Recados administrados.
- Logs minimizados.
- UI nao pode ser unica permissao.

Nao fazer: copiar Admin setup legado, segredo legado, roles so no frontend ou log sensivel bruto.

## Fase 15 - QA

- Seed demo como fixture.
- Dados extremos.
- Textos longos.
- Itens incompletos.
- Itens atrasados.
- Dependencias.
- Anexos.
- Unknown.
- Alta densidade.
- Mobile.
- Safe area.
- Overflow.
- Empty states.
- Erro.

Nao fazer: validar componente so com mock ideal.
